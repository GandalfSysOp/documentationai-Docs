---
title: How to add and manage custom roles
---
ProofHub offers a flexible role management system to help you control access and permissions for team members. While there are three default roles (Owner, Admin, and Normal User) that cannot be customized, you have the option to create custom roles tailored to your specific needs.

## **Default roles overview**

- **<u>Owner:</u>** Manages all projects (create, edit, delete), controls user access (admin/user), sees all projects, and has unrestricted access to private items (tasks, notes, timesheets). The Owner is the administrator with full control.

- **<u>Admin:</u>** Users with administrative privileges, but not the same level of control as the Owner.

- **<u>Normal user:</u>** Standard users have access to their assigned tasks and projects but limited administrative capabilities.

1. **Manage section**: Select the “**Manage”** section and select “**Custom roles**“.\

2. Click on the **“Add”** button.\

3. In the add role window, enter the role title and a description\


   <Image src="https://help.proofhub.com/swift/wp-content/uploads/2024/05/Screenshot-2024-05-20-at-11.32.09-AM.png" width="712" height="311" alt="" />

## **Create a copy**

1. Right-click or click on the three dots and select **“Create a Copy”**.\


   <Image src="https://help.proofhub.com/swift/wp-content/uploads/2024/05/Screenshot-2024-05-20-at-11.29.08-AM.png" width="1047" height="701" alt="" />

2. In the copy window, rename the role title if necessary.\


   <Image src="https://help.proofhub.com/swift/wp-content/uploads/2024/05/Screenshot-2024-05-20-at-11.18.05-AM.png" width="715" height="310" alt="" />

3. Click “**Ok**“.

## **Delete custom role**

1. Right-click or click on the three dots and select **“Delete”**.\


   <Image src="https://help.proofhub.com/swift/wp-content/uploads/2024/05/Screenshot-2024-05-20-at-11.33.57-AM.png" width="1041" height="706" alt="" />

2. **Assign another role:** In the confirmation window, select the new role you want to assign to the users currently holding the role you are deleting.\


   <Image src="https://help.proofhub.com/swift/wp-content/uploads/2024/05/Screenshot-2024-05-20-at-11.18.56-AM.png" width="500" height="241" alt="" />

## **Access levels for different sections**

### **Project**

- **<u>View only</u>****:** Users with this access level can only view projects. They cannot add, edit, delete, manage people, or duplicate projects.

- **<u>View</u>**<u> </u>**<u>and manage</u>**<u>:</u> This option allows users to add, edit, delete, manage people, and copy projects.\

**View projects option:**

This option lets you restrict someone to viewing assigned projects or give access to all projects. The available options are:

- **<u>All projects</u>****:** With this access level, the user can view all projects.

* **<u>Assigned projects</u>****:** With this access level, the user can only view projects assigned to them.\

  <Image src="https://ci3.googleusercontent.com/meips/ADKq_NYwFJTaNzNFES52gCnmUlQgjR8h-baFKuvjvj2rZXz19yI02HM2Ctmy57H4iSGq66-8LVBbF_kd7q-EI3V4nojRBzctVzISWp9PFTWNWM4a_L3k77w=s0-d-e1-ft#https://help.proofhub.com/plus/wp-content/uploads/2018/10/7.png" width="894" height="212" alt="7" />

To let someone manage categories, select the “**Manage Categories**” checkbox. To allow management of project statuses, select the “**Manage project statuses**” checkbox.

### **Discussions**

- **<u>View only:</u>** A user can only view discussion topics, no edits or additions.

* **<u>View and add</u>****:** A user can view and add new discussion topics.

- **<u>View and manage</u>****:&#xA0;**&#x57;ith this access level, a user can start, edit, delete, and manage discussion topics.

Here are the access levels for files and comments on a topic:

- **View only**: A user can view files and comments but cannot upload, delete, rename, or manage files.

* **View and add**: Using this access level. Users can view and add files to the topic and comments.

- **View and manage**: Can upload, delete, rename, and manage files.

**File downloading**: Select the checkbox to allow file downloads to users.

**File proofing**: If you want to allow users to proof files, then you can select the checkbox “**Allow file proofing**” option.\


<Image src="https://help.proofhub.com/plus/wp-content/uploads/2018/10/11-2.png" width="704" height="274" alt="11" />

### **Tasks**

- **<u>View only</u>****:** With this access level, the user can view tasks but cannot add, edit, delete, comment, or archive tasks.

* **<u>View and add</u>****:** Users can view and add task lists and tasks.

- **<u>View and manage</u>****:** With this access level, users can view, add, assign, edit, delete, copy, and move task lists and tasks.



<Image src="https://help.proofhub.com/plus/wp-content/uploads/2018/10/12-1.png" width="884" height="251" alt="12" />

Access levels for task comments:

**<u>View only</u>****:** With this access level, a user can only view the comments on the task and cannot add, edit, or delete a comment.

**<u>View and add</u>****:** Users can only view the comments of others and add comments using this access.

**<u>View and manage</u>****:** A user can post, edit, and delete both their own and others’ comments.



<Image src="https://help.proofhub.com/plus/wp-content/uploads/2018/10/13.png" width="858" height="312" alt="13" />

Task completion and stage change access levels:

- **<u>Assigned to anyone and unassigned</u>****:** A user can complete or change the stage of any task, regardless of assignment.

* **Assigned to them/unassigned**: Can complete tasks assigned to them or unassigned tasks.

- **<u>Assigned to them and unassigned:</u>** With this access level, a person can complete the task assigned to him and also those tasks that have not been assigned to anyone.

* **<u>Assigned to them</u>****:** A user can complete only their assigned tasks.

  <Image src="https://help.proofhub.com/plus/wp-content/uploads/2018/10/16.png" width="887" height="327" alt="16" />

Access levels for task file attachments:

- **<u>View only</u>****:** With this access level a user can only view files attached to tasks. The user cannot upload a new file, a new version of an existing file, rename, delete, move, etc.

* **<u>View and add:</u>****&#xA0;**&#x43;an view and attach new files to tasks.

- **<u>View and manage</u>****:** Users can view, add, rename, update, move, and delete files.

Choose the “**View all tasks**” checkbox to access the All Tasks tab and the “**View Gantt**” checkbox to manage the task list in the Gantt chart.

Select the checkboxes for “**Allow file proofing**,” “**Manage workflows**,” and “**Manage task labels**” to grant access for proofing attached files, managing **workflows**, and managin&#x67;**&#x20;task labels**, respectively.\


<Image src="https://help.proofhub.com/plus/wp-content/uploads/2018/10/14.png" width="838" height="209" alt="14" />

### **Calendar**

- **<u>View only</u>****:** With this access level, a user can only view items in the calendar and cannot create, edit, or delete milestones, events, or tasks.

* **<u>View and add</u>****:** With this access level, a user can view existing items and add tasks, milestones, and events to the calendar.

- **<u>View and manage</u>****:** A user can create, edit, and delete milestones, events, and tasks using this access level.



<Image src="https://help.proofhub.com/plus/wp-content/uploads/2018/10/18.png" width="871" height="257" alt="18" />

Access levels for completion of milestones:

- **<u>Assigned to anyone and unassigned:</u>** This access level allows a user to complete milestones regardless of their assigned owner.

* **<u>Assigned to them and unassigned:</u>** With this access level, a user can complete milestones assigned to them as well as those that are unassigned.

- **<u>Assigned to them</u>****:** A user can only complete milestones that are assigned to them.



<Image src="https://help.proofhub.com/plus/wp-content/uploads/2018/10/19.png" width="875" height="255" alt="19" />

If you want to grant any user to view the **All calendars** tab then select the checkbox “**View All calendars”&#xA0;**&#x67;iven in this section.

To allow any person to subscribe to ProofHub’s calendar to any third-party tool like Outlook then you can select checkboxes with “**Outlook”** and “**Allow subscribing to ProofHub calendar**“.\


<Image src="https://help.proofhub.com/plus/wp-content/uploads/2018/10/21.png" width="872" height="240" alt="21" />

### **Notes**

- **<u>View only</u>****:** With this access level, a user can only view notes and cannot create notebooks or notes, edit or delete notes, post comments, or manage collaborators.

* **<u>View and add</u>****:** Using this access level, a user can create notebooks and can add notes to them.

- **<u>View and manage</u>****:** A user with this access level can create notebooks, add notes, manage collaborators, edit and delete notes, and post comments.



<Image src="https://help.proofhub.com/plus/wp-content/uploads/2018/10/23.png" width="872" height="250" alt="23" />

Access levels for the content of notes:

- **<u>All notes</u>****:** With this access level a user will be able to manage the content of all notes irrespective of the fact that who has added it.

* **<u>Self-added notes</u>****:** With this access level, a user will be able to edit and manage the content that has been added by himself.

**<u>Note:</u>** *With this access level, a person will NOT be able to manage anybody’s content. He will only have read-only access to the content.*\


<Image src="https://help.proofhub.com/plus/wp-content/uploads/2018/10/24.png" width="865" height="243" alt="24" />

### **Files**

- **<u>View only</u>****:** With this access level, the user can view files but will not have the ability to add, rename, move, or delete files, create folders or subfolders, or upload new versions.

* **<u>View and add</u>****:&#xA0;**&#x55;sers can view files and folders, add content to the **Files** section of a project, and create new folders and files.

* **<u>View and manage</u>****:** With this access level, a user can also upload files and create folders and subfolders.\


  <Image src="https://help.proofhub.com/plus/wp-content/uploads/2018/10/26.png" width="756" height="249" alt="26" />

To grant file downloading access, select the “**Allow file downloading**” checkbox and to enable file proofing, select the “**Allow file proofing**” checkbox.

### **Timesheets**

Following are the access levels for the timesheet:

- **<u>View only</u>****:** With this access level, the user can view and download timesheets in CSV format but cannot add log time, or archive timesheets.

* **<u>View and add</u>****:** The users can view, add, and record time in a timesheet. If the **“Allow to edit and delete self-added data”** option is enabled, they will also be able to edit and delete only the timesheets and time entries they have created.

* **<u>View and manage: </u>**&#x57;ith this access level, the user can create, edit, delete, and archive timesheets, as well as add, edit, and delete time entries.

To allow a person to view the “All-time” tab in the Everything section, select the “**View All Time**” checkbox.\


<Image src="https://help.proofhub.com/plus/wp-content/uploads/2018/10/28.png" width="701" height="234" alt="28" />

To enable exporting time entries to FreshBooks, select the “**Export time entries to FreshBooks**” checkbox.

### **Chat**

- **<u>Allow</u>****&#x20;****<u>access</u>****:&#xA0;**&#x42;y selecting this option you can enable and disable chat for people

* **<u>Allow group chat</u>****:** By selecting this option you can allow one to do group chat with people visible to you by creating a group.

**<u>Note:</u>** *If you’ve restricted the users to only assigned projects, then they will be able to chat only with the people who are added to them in their assigned projects.*

Granting access to “All projects” provides the option to select “Show people” from either “All projects” or “Assigned project.” If you choose “All projects,” all users across projects will be visible in chat.\


<Image src="https://help.proofhub.com/plus/wp-content/uploads/2018/10/32.png" width="864" height="217" alt="32" />

### **Announcements**

- **<u>View only</u>****:&#xA0;**&#x57;ith this access level one will only be able to see announcements and will not be able to add, edit, or delete an announcement.

* **<u>View and add:</u>****&#xA0;**&#x55;sing this access level, the user can only add announcements. If the **“Allow to edit and delete self-added data”** option is selected, they can also edit and delete their own announcements.

- **<u>View and manage</u>****:&#xA0;**&#x55;sing this access, the user can add, edit, and delete all announcements, regardless of who created them.



<Image src="https://help.proofhub.com/plus/wp-content/uploads/2018/10/33.png" width="864" height="205" alt="33" />

Access levels for comments in announcements:

- **<u>View only</u>****:** With this access level one will only be able to see comments made in an announcement. One will not be able to add a comment, edit a comment, or delete the comment.

* **<u>View and add</u>****:** With this access level, one will be able to add comments on announcements. In case **Allow to edit and delete self-added data** is selected then one will be able to edit and delete comments made by them on announcements.

- **<u>View and manage</u>****:&#xA0;**&#x57;ith this access level, one will be able to add, edit, and delete any comment added in an announcement.



<Image src="https://help.proofhub.com/plus/wp-content/uploads/2018/10/34.png" width="859" height="199" alt="34" />

### **Reports**

- **<u>View only</u>****:&#xA0;**&#x57;ith this access level, one will be able to view all existing custom reports. One won’t be able to add a new report, edit existing reports, or delete a report.

* **<u>View and add</u>****:&#xA0;**&#x57;ith this access level one will be able to view all existing reports and one will be able to create a new custom report. In case “**Allow to edit and delete self-added data**” is selected then one will be able to edit and delete custom reports made by them.

- **<u>View and manage</u>****:** With this access level, one will be able to view, edit, and delete all existing reports. Also, one will be able to create new custom reports.

### **Others**

This section allows one to grant access to the other settings of the application.

- **<u>Manage</u>****&#x20;****<u>company, custom domain & preferences:</u>** You can select the checkbox of “Manage company, custom domain & preferences” if you want to provide access to ProofHub settings to any person from their account.

* **<u>Manage people and groups:</u>** Similarly, you can use this option if you wish to give them access to managing people and groups created in the “**People section**” and also to add new people to your account.

- **<u>Manage plans & billing:</u>** With this option, you can provide users access to making payments or see invoices of the payment made previously.

* **<u>Export data from ProofHub</u>****:** With this option, you can allow a person to export data from your ProofHub account.

- **<u>View logs:</u>** With this option, you can give users access to view activity logs from their accounts.

* **<u>Manage apps:</u>** With this option, you can give users access to manage third-party apps (install and uninstall).

- **<u>Allow to edit and delete self-added data</u>****:** This option is related to the **“view and add**” access level. If you want to give a person access to edit the data added by himself or herself, then you can select this option.

**<u>Manage trashed items:</u>**<u> </u>With this option, you can give access to the trash where one can manage items deleted.\

* **<u>Allow access to Box files:</u>** With this option, one can provide access to attaching files from Box.

- **<u>Allow access to OneDrive files</u>****:** With this option, one can provide access to attaching files from OneDrive.\
