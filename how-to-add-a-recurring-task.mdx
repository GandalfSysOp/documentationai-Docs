title: How to add a recurring task
---
## **Introduction**

ProofHub offers a convenient feature to create recurring tasks, allowing you to automate the scheduling of tasks that repeat daily, weekly, monthly, or yearly.

## **Add a recurring task**

1. **<u>Create a new task:</u>** First, begin by creating a new task in your ProofHub project. If you’re unsure how to create a task, refer to this <a href="https://help.proofhub.com/swift/tasks/how-to-add-and-manage-task" target="_blank" rel="noopener noreferrer">link</a> for detailed instructions.

2. **<u>Enable recurrence:</u>** As you create your task, ensure you set both the start date and due date. The due date is particularly important for defining the recurrence pattern. After setting the dates, click on the “**Repeat**“.


   <Image src="https://help.proofhub.com/swift/wp-content/uploads/2024/05/Screenshot-2024-05-16-at-9.27.51-AM.png" width="718" height="572" alt="" />

3. **<u>Recurrence frequency:</u>** Select the desired recurrence frequency – *Daily, Weekly, Monthly, or Yearly.*


   <Image src="https://help.proofhub.com/swift/wp-content/uploads/2024/05/Screenshot-2024-05-16-at-9.29.26-AM.png" width="715" height="610" alt="" />

4. **<u>Configure recurrence pattern:</u>** Once you’ve selected the recurrence frequency, the new window will specify the details of the pattern.


   <Image src="https://help.proofhub.com/swift/wp-content/uploads/2024/05/Screenshot-2024-05-16-at-9.31.57-AM.png" width="375" height="415" alt="" />

5. **<u>Apply changes:</u>** Click on the “**Apply**” button.

**<u>Notes:</u>**

1. **<u>Handling task completion and due dates:</u>** Should a task recur every third day and its original due date be the 10th, but if, for any reason, it is marked as complete on the 11th, the subsequent task will then be due on the 14th of the month.

2. **<u>Moving tasks between projects:</u>** If you move a task list containing recurring tasks to another project within ProofHub, any remaining instances of the recurring tasks will be created in the new project according to their recurrence settings.

3. **<u>Copying task lists:</u>** When you copy a task list containing recurring tasks to another project, please note that the remaining instances of the recurring tasks will not be created in the new project. This is because due dates are not copied over during the duplication process.
