---
title: Import projects from Asana
---
Already using Asana? Try ProofHub with your existing data!

ProofHub offers the ability to import your projects directly from Asana. If you’re already a user of Asana with existing projects, our import feature enables you to migrate people, projects, tasks, files, and more seamlessly. Here’s how it works:

1. **<u>Navigate to the manage section</u>****:&#xA0;**&#x43;lick on the gear icon on the left side. Select “**View logs > Import”**.\


   <Image src="https://help.proofhub.com/swift/wp-content/uploads/2024/05/11-1.png" width="562" height="778" alt="" />



2. **<u>Import from Asana</u>****:&#xA0;**&#x43;lick on the **“Import”** button in front of the Asana option and log into your Asana account.\


   <Image src="https://help.proofhub.com/swift/wp-content/uploads/2024/05/12-2.png" width="1432" height="492" alt="" />



3. **<u>Select workspace</u>****:&#xA0;**&#x53;elect the workspace from where you want to import the projects.\


   <Image src="https://help.proofhub.com/swift/wp-content/uploads/2024/05/13-1.png" width="538" height="329" alt="" />

4. **<u>Select projects</u>****:&#xA0;**&#x53;elect all or multiple projects to be imported.\


   <Image src="https://help.proofhub.com/swift/wp-content/uploads/2024/05/14-1.png" width="684" height="512" alt="" />

5. **<u>Invite people</u>****:** You can send invitations to people while importing projects to ProofHub by **“Send invitation to all imported contacts”**.\


   <Image src="https://help.proofhub.com/swift/wp-content/uploads/2024/05/15-1.png" width="685" height="143" alt="" />

6. All the projects will be imported into a default project category called **“Asana”** and all the people will be imported into a group with the same name as in Asana.\


   <Image src="https://help.proofhub.com/swift/wp-content/uploads/2024/05/16-1.png" width="1209" height="475" alt="" />

   \
   Once the import is done, you’ll receive an email notification of the confirmation of import.

**<u>Note:</u>****&#xA0;***All the imported tasks will come up in the very first stage of the selected workflow.*
